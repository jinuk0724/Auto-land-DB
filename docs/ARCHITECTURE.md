# 전국 부동산 인사이트 데스크톱 앱 아키텍처

## 기술 스택 확정

연결된 Windows 컴퓨터에는 Node.js와 .NET SDK가 설치되어 있지 않고 Git만 확인되었습니다. 따라서 전역 설치나 관리자 권한 의존을 최소화하기 위해 프로젝트 폴더 내부에 **portable Node.js**를 내려받아 사용하고, 앱은 **Electron + Vite + TypeScript** 기반의 데스크톱 애플리케이션으로 구성합니다. Electron은 Windows 데스크톱 앱으로 패키징할 수 있고, Chromium 기반 UI를 제공하므로 지도·차트·검색형 대시보드를 빠르게 구현하기 적합합니다.

| 구분 | 선택 | 이유 |
| --- | --- | --- |
| 데스크톱 런타임 | Electron | Windows에서 별도 .NET SDK 없이 Node 기반으로 빌드 가능하며 지도·차트 UI 구현이 쉽습니다. |
| UI 빌드 | Vite + TypeScript | 빠른 개발 서버와 정적 번들 생성이 가능하고 유지보수가 쉽습니다. |
| 지도 | Leaflet + OpenStreetMap | 별도 지도 API 키 없이 전국 지도 시각화가 가능합니다. |
| 차트 | Chart.js | 실거래·상권·유동인구 지표를 간단히 시각화할 수 있습니다. |
| API 통신 | Electron main process IPC | 공공데이터포털 API의 CORS·키 보관 문제를 줄이기 위해 메인 프로세스에서 호출합니다. |
| 데이터 저장 | localStorage 기반 설정 + 샘플 데이터 | 초기 버전에서는 API 키와 최근 검색 조건을 로컬에 저장하고, 추후 SQLite 확장을 고려합니다. |

## 기능 범위

초기 빌드에서는 사용자가 **공공데이터포털 서비스키**를 입력하면 실제 API 요청을 수행하고, 서비스키가 없거나 네트워크 오류가 발생하면 샘플 데이터로 UI와 분석 흐름을 확인할 수 있게 합니다. 실시간 민간 매물 크롤링은 약관 및 법적 리스크가 있어 제외하고, 공공 실거래·공매·상권 데이터 중심의 분석 앱으로 구현합니다.

| 기능 | 초기 구현 방식 | 향후 확장 |
| --- | --- | --- |
| 주거용 전월세 검색 | 국토부 아파트 전월세 API 연동 구조 + 샘플 데이터 | 아파트 매매, 연립다세대, 단독다가구, 오피스텔 API 추가 |
| 비주거용 매매 검색 | 국토부 상업업무용 매매 API 연동 구조 + 샘플 데이터 | 토지·오피스텔 매매 API 추가 |
| 경매/공매 검색 | 온비드 부동산 물건목록 API 연동 구조 + 법원경매 바로가기 | 유료 법원경매 API 또는 사용자 승인 기반 수집 기능 |
| 상권 분석 | 소상공인 상가정보 API/샘플 기반 업종 분포, 경쟁밀도 분석 | 전국 CSV 인덱싱, 반경·폴리곤 분석 |
| 유동인구 분석 | 데모 지표와 상권 점수 계산 로직 제공 | 서울시 생활인구·상권분석 데이터 및 통신사/민간 API 연동 |
| 지도 시각화 | Leaflet 마커·클러스터형 지도 | 히트맵, 행정동 경계, 권역별 색상 지도 |

## 화면 구성

앱은 좌측의 검색·필터 패널, 중앙의 지도, 우측의 분석 요약 패널로 구성합니다. 하단에는 검색 결과 테이블과 상세 카드가 배치됩니다. 사용자는 지역, 거래 유형, 부동산 유형, 계약월, 가격 범위, 반경을 선택하고 결과를 확인할 수 있습니다.

| 영역 | 주요 UI |
| --- | --- |
| 상단 헤더 | 앱명, API 키 설정, 데이터 출처 표시, 법원경매 바로가기 |
| 좌측 패널 | 지역, 거래유형, 물건유형, 계약년월, 가격/면적 필터, 검색 버튼 |
| 중앙 지도 | 실거래/공매/상가 마커, 선택한 물건 위치, 반경 표시 |
| 우측 패널 | 평균 거래가, 월세/전세 지표, 업종 분포, 유동인구 점수, 투자 위험도 |
| 하단 테이블 | 결과 목록, 정렬, CSV 내보내기 |

## 데이터 연동 설계

Electron 메인 프로세스는 `realEstateApi.ts`에서 데이터 소스별 API 호출 함수를 제공합니다. 렌더러는 `window.api.searchRealEstate()` 형태의 안전한 preload 브리지를 통해 검색 요청을 보내고 결과만 받습니다. API 키는 사용자의 로컬 PC에 저장되며 외부 서버로 전송하지 않습니다.

> 초기 버전의 핵심 목표는 “실제 API 키를 넣으면 공공데이터를 조회할 수 있는 데스크톱 앱 골격”과 “API 키 없이도 전체 분석 UX를 검증할 수 있는 샘플 기반 데모”를 동시에 제공하는 것입니다.
